โรงเรียนอนุบาลหมีน้อย มีนักเรียนเป็นจำนวนมาก ทำให้เวลาเช็คชื่อในแต่ละคาบใช้เวลาค่อนข้างนาน
ทางผู้บริหารของโรงเรียนจึงเล็งเห็นถึงปัญหานี้ และได้ให้ทีมพัฒนาระบบเข้ามาปรับปรุงระบบในการเช็คชื่อให้มีประสิทธิภาพมากขึ้น<br><br>

ผู้พัฒนาระบบได้ทำการศึกษา requirements ของระบบ และได้แบ่งระบบเวบไซต์เช็คชื่อออกเป็น 3 ส่วนหลักๆ ได้แก่  ส่วนเช็คชื่อ ส่วนออกรายงาน และส่วนจัดการข้อมูลนักเรียนและวิชาเรียน<br><br>

  ส่วนเช็คชื่อมีหน้าจอดังนี้<br>
    1. หน้าจอรายการวิชาที่มีการสอนในวันนั้น ๆ ของ อ. ที่ login เข้ามา<br>
    2. หน้าจอรายละเอียดของแต่ละวิชา (วิชานี้สอนอะไร, มีจำนวนนักเรียนกี่คน, มีคนมาเรียน และขาดกี่คน) - ยกตัวอย่าง endpoint "/course/{id}/"<br>
    3. หน้าจอเช็คชื่อของแต่ละวิชาที่สามารถเช็คชื่อได้ด้วย QR code<br><br>

  ส่วนออกรายงานมีหน้าจอดังนี้<br>
    1. หน้าจอ dashboard สามารถดูภาพรวมของระบบได้ว่า ในวันนั้น ๆ แต่ละวิชามีจำนวนคนเข้าเรียนกี่คน ขาดกี่คน<br>
    2. หน้าจอค้นหา และ export ข้อมูลการเข้าห้องเรียน ทั้งในเทอมปัจจุบัน และ ย้อนหลัง<br><br>

  ส่วนจัดการข้อมูลนักเรียนและวิชาเรียน<br>
    1. หน้าจอรายการนักเรียนทั้งหมด<br>
    2. หน้าจอเพิ่มนักเรียน<br>
    3. หน้าจอแก้ไขข้อมูลนักเรียน<br>
    4. หน้าจอรายการวิชาเรียนทั้งหมด<br>
    5. หน้าจอเพิ่มวิชาเรียน<br>
    6. หน้าจอแก้ไขข้อมูลวิชาเรียน<br><br>
    
 โดยในสามารถเข้าหน้าเว็บได้ 3 ส่วนตาม path ที่กำหนดได้ดังนี้<br>
 1.หน้าเช็คชื่อสามารถเข้าได้ผ่าน /course/course_id<br>
 2.หน้าออกรายงานสามารถเข้าได้ผ่าน /display<br>
 3.หน้าจัดการข้อมูลนักเรียนสามารถเข้าได้ผ่าน /manage<br>
